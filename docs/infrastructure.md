# Client distribution infrastructure

The hosted Savecraft product is proprietary and lives outside this repository.
This document covers the public client's release and distribution surfaces.

## Distribution Worker

[`install/worker`](../install/worker/) is the repository's sole hosted
exception. It serves the installer entry point and publishes client artifacts
through Cloudflare. It does not parse or store game saves.

The install bucket remains private. The Worker mediates reads for installer,
daemon, tray, MSI, signature, checksum, and update-manifest artifacts.

## Release workflows

The workflows under [`.github/workflows`](../.github/workflows/) have separate
release namespaces:

| Component | Tag | Workflow |
|---|---|---|
| Daemon, tray, MSI | `daemon-v*` | `deploy-daemon.yml` |
| Installer and distribution Worker | `install-v*` | `deploy-install.yml` |
| Parser plugin | `plugin-{game_id}-v*` | `deploy-plugin.yml` |

Staging versions use the CI run number and commit; production versions come
from component tags.

## Daemon artifacts

The daemon workflow cross-compiles Linux, macOS, and Windows binaries, builds
the Windows tray and MSI, signs release artifacts, emits checksums, and uploads
a signed daemon update manifest. The self-updater verifies the manifest and
artifact signatures before replacing a binary.

## Plugin artifacts

The plugin workflow builds only public client halves. For WASM games it signs
and uploads `parser.wasm`; mod-only games publish metadata and their separately
packaged in-game mod. `cmd/plugin-registry` produces `client-manifest.json`.

After per-game publication, the workflow downloads every current client
descriptor, composes the complete aggregate, writes its bytes once, signs those
literal bytes, and uploads the same file as `plugins/manifest.json` with its
detached signature. The daemon pins download URLs to the configured server
origin.

## Protocol generation

Go protobuf output is generated by the pinned Buf plugin in `buf.gen.yaml`.
RimWorld C# output is generated with protoc from the Nix/devenv toolchain. CI
runs both recipes and requires a clean diff, preventing generator drift.

## Signing key

Release workflows receive the Ed25519 private key from GitHub Actions secrets.
Only the public key is checked into the repository. Unsigned or modified
manifests, parser modules, and daemon artifacts are rejected by the client.
